WINDOWS 10 / 11 PORTABLE QUICK START

1. Extract the complete ZIP to a short writable path.
2. Double-click THEKEY.exe.
3. Choose "Seleccionar y analizar / Select & Analyze" and select a project folder.
   Use SAMPLE-PYTHON-APP for a healthy verification or
   SAMPLE-REPAIRABLE-PYTHON-APP for a transparent repair scenario.
4. The first analysis is read-only and does not execute selected-project code.
5. Choose "Verificar / Verify" only after explicitly consenting to trusted
   tests in an isolated copy. The original project is not modified.
6. Choose "Reparar / Repair" to produce a bounded repair preview in isolation.
   Review its real output. Applying it needs a separate confirmation, a fresh
   source hash check, an external backup, and post-apply re-verification.
7. Choose "Demo para jueces / Judge demo" for the deterministic Build Week
   scenario, then inspect persisted evidence in "Ver resultados / View results".

SECURITY BOUNDARY

Project tests are trusted local code. THEKEY protects the original through
read-only intake and an isolated working copy, but this is not an operating-
system sandbox. Do not consent to tests from an untrusted project.

The home screen intentionally shows no fabricated health metrics or activity.
Results and activity appear only after THEKEY has produced local evidence.

The portable application requires no Python, Git, PowerShell, API key, Docker,
WSL, GPU, paid service, or test account. Do not move THEKEY.exe out of its
extracted folder. BUILD_MANIFEST.json records the source state and SHA-256 of
distributed files.
