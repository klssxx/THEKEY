## Context
The primary Quick Start runs on Windows 11 from a path that may contain spaces. `scripts/demo.ps1` already sets `THEKEY_REPO_ROOT` to isolate run state, but other auxiliary scripts (bootstrap, clean, run-demo) may not be space-safe.

## Requisito o riesgo al que responde
- Requisito: THEKEY_CONTRACT §1 (isolated workspace; the run must work wherever the clone lives) and §5 (reproducible audit).
- Riesgo cubierto: THREAT-T1 (write outside the isolated workspace) — a path-with-spaces bug can resolve `cwd` outside the clone and leak/touch the wrong tree.

## Impacto si no se resuelve
A contributor who clones to `C:\Users\Name\My Projects\THEKEY` hits a silent failure or, worse, a wrong-path write — breaking the "works from any Windows path" promise.

## Acceptance criteria
- [ ] All auxiliary scripts handle `THEKEY_REPO_ROOT` and spaces in paths (subprocess list args, not shell strings).
- [ ] A CI matrix or local scenario runs the full Quick Start from a path with spaces and reaches `RELEASE_ELIGIBLE`.
- [ ] No script assumes a space-free `cwd`.

## Cómo se verifica
```powershell
# clone to a path with spaces, then:
pwsh -NoProfile -File .\scripts\demo.ps1
# expected: exit 0, RELEASE_ELIGIBLE, gates_passed: 4/4, workspace under the spaced clone
```

## Evidencia en event store
On merge record `Closes #6` + the spaced-path demo output in the PR body; `evidence_mismatches: []` confirms the run resolved to the clone, not the host checkout.
